# BizLP Website - プロジェクト構造詳細ガイド

## 📋 プロジェクト概要
Next.js + TypeScript + Tailwind CSS + microCMS で構築されたビジネスランディングページ

---

## 🎯 画面ファイル（実際のページ）

### `/pages/` フォルダ
Next.jsのファイルベースルーティングにより、各ファイルがURLに対応

| ファイル | URL | 役割 | 詳細 |
|---------|-----|------|------|
| `index.tsx` | `/` | **ホームページ** | サイトのトップページ。最新ニュース5件を表示、CTAボタン配置 |
| `news/index.tsx` | `/news` | **ニュース一覧** | 全ニュース記事の一覧表示、ページネーション対応 |
| `news/[id].tsx` | `/news/記事ID` | **ニュース詳細** | 個別記事の詳細表示、関連記事の表示 |
| `company.tsx` | `/company` | **会社概要** | 企業情報、沿革、理念などの静的コンテンツ |
| `contact.tsx` | `/contact` | **お問い合わせ** | フォーム送信機能、送信完了メッセージ表示 |
| `privacy.tsx` | `/privacy` | **プライバシーポリシー** | 個人情報保護方針の法的文書 |
| `terms.tsx` | `/terms` | **利用規約** | サイト利用に関する規約・免責事項 |
| `_app.tsx` | - | **アプリケーションラッパー** | 全ページ共通の設定、グローバルスタイル適用 |

## 🧩 コンポーネント・ライブラリ

### `/components/` フォルダ

| `Layout.tsx` | **共通レイアウト** | ヘッダー・ナビゲーション・フッターを含む全ページ共通のレイアウト。SEO設定（title, description）も担当 |

| ファイル | 役割 | 詳細 |
|---------|------|------|
| `microcms.ts` | **API通信クライアント** | microCMSとの通信を担当。ニュースデータの取得・管理、TypeScript型定義、エラーハンドリング |

## ⚙️ 設定ファイル

### Next.js関連
| ファイル | 役割 | 詳細 |
|---------|------|------|
| `next.config.js` | **Next.js設定** | 画像最適化（microCMS画像ドメイン許可）、React Strict Mode、ビルド設定 |

## 🏗️ アーキテクチャパターン

### 1. **ファイルベースルーティング**
- `/pages/` フォルダ内のファイル構造がURL構造に対応
- `[id].tsx` で動的ルーティング実現

### 2. **コンポーネント分離**
- 再利用可能なUIコンポーネントを `/components/` に分離
- ビジネスロジックを `/lib/` に分離

### 3. **型安全性**
- TypeScriptによる厳密な型チェック
- APIレスポンスの型定義

### 4. **レスポンシブデザイン**
- Tailwind CSSによるモバイルファーストデザイン
- カスタムブレークポイント設定

### 5. **SEO最適化**
- 各ページでのメタタグ設定
- 構造化データ対応

---

## 🔄 データフロー

```
microCMS → lib/microcms.ts → pages/*.tsx → components/Layout.tsx → ブラウザ
```

1. **データ取得**: microCMSからAPI経由でデータ取得
2. **型変換**: TypeScript型でデータを型安全に処理
3. **ページ表示**: Next.jsのSSG/SSRでページ生成
4. **レイアウト適用**: Layoutコンポーネントで共通UI適用
5. **ユーザー表示**: ブラウザで最終的なページ表示

## 🚀 開発・デプロイフロー

### 開発環境
```bash
npm run dev    # 開発サーバー起動 (localhost:3000)
npm run lint   # コード品質チェック
```

### 本番環境
```bash
npm run build  # 本番用ビルド
npm run start  # 本番サーバー起動
```
